  Class: CRMWorker
    - __init__(storage: SupabaseStorage, memory: ZylchMemory, anthropic: Anthropic)

    - async compute_batch(contact_ids: List[str], owner_id: str) -> None
    - _compute_status(last_email_from_owner: bool, memory_context: List) -> str

  STATUS LOGIC (must match plan exactly):
  - "open": last_email_from_owner=False AND no "no response" in memory
  - "waiting": last_email_from_owner=True
  - "closed": "no response" pattern in memory OR manually closed

  PRIORITY FORMULA (1-10):
    urgency = 4 if days_since > 7 else (2 if days_since > 3 else 0)
    importance = int(relationship_strength * 2) + int(topic_importance * 2)
    priority = min(10, max(1, 2 + urgency + importance))

  ACTION GENERATION:
  - Use Claude Haiku (claude-3-5-haiku-20241022)
  - Max 100 tokens, 80 char output
  - Specific actions only (not vague "follow up")
  - Return None if status == "closed"

  PHASE 4: INTEGRATION (Agent 3, Week 5)
  ═══════════════════════════════════════════════════════════════

  FILE: zylch/services/sync_service.py

  MODIFY: async def run_full_sync(owner_id: str, days_back: int = 30)

  NEW FLOW (sequential pipeline):
  1. await sync_emails(owner_id, days_back)  # Existing
  2. # NEW: Memory Agent
     memory_worker = MemoryWorker(storage, zylch_memory)
     new_emails = storage.get_unprocessed_emails(owner_id)
     await memory_worker.process_batch(new_emails, owner_id)
  3. # NEW: CRM Agent
     crm_worker = CRMWorker(storage, zylch_memory, anthropic_client)
     affected_contacts = storage.get_affected_contacts([e['id'] for e in new_emails])
     await crm_worker.compute_batch(affected_contacts, owner_id)
  4. display_gaps(owner_id)  # Existing

  PERFORMANCE TARGET: <20s for 30-day window (10 emails)
  - Email sync: 1-2s
  - Memory Agent: ~10s (batched)
  - CRM Agent: ~5s (parallel)
  - Display: 1s

  ERROR HANDLING:
  - If Memory Agent fails: log error, continue to CRM with existing Memory
  - If CRM Agent fails: log error, Avatar remains stale but /sync completes
  - Never crash the entire /sync pipeline

  LOGGING:
  - INFO: "Memory Agent: Processed 10 emails in 9.2s"
  - INFO: "CRM Agent: Computed 7 avatars in 4.8s"
  - DEBUG: Individual extraction/computation details
